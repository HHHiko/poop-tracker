# 需求定义文档

## 项目名称

便便记录助手

## 项目背景

用户需要一个简单直观的工具来记录每日排便情况，替代传统的文字备忘录方式。目标是通过图标化、可视化的方式，让记录行为变得快速轻松。

## 用户画像

- 不懂代码的普通用户
- 希望用手机快速记录，不要繁琐输入
- 关心自己的肠道健康规律
- 需要直观看到一段时间的趋势

## 功能需求

### F1 — 月历页（首页）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1.1 | 显示当月日历网格（周日~周六） | P0 |
| F1.2 | 左右箭头切换上/下月份 | P0 |
| F1.3 | 每个日期格子内用彩色小圆点标记当天的便便记录 | P0 |
| F1.4 | 不同颜色圆点对应不同便便类型 | P0 |
| F1.5 | 当天有多条记录时显示多个圆点 | P1 |
| F1.6 | 点击某个日期 → 跳转日详情页 | P0 |
| F1.7 | 浮动 "+" 按钮 → 跳转记录页（默认日期为今天） | P0 |
| F1.8 | 今天日期高亮显示 | P1 |

### F2 — 记录页

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F2.1 | 5 种便便颜色（棕/绿/黄/深/红），以大图标卡片展示，点击选中 | P0 |
| F2.1a | 每种颜色附带简短健康说明（如：棕色=健康正常，红色=需留意） | P1 |
| F2.2 | 便便类型卡片：选中态有明显视觉反馈（边框高亮/颜色变化） | P0 |
| F2.3 | 时间选择器：点击弹出微信原生时间选择器 | P0 |
| F2.4 | 可选备注输入框（默认折叠，点击展开） | P1 |
| F2.5 | 保存按钮 → 数据存入本地 → 返回上一页 | P0 |
| F2.6 | 保存时校验：必须选择便便类型和时间 | P0 |

### F3 — 日详情页

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F3.1 | 显示选中日期的所有记录（按时间排序） | P0 |
| F3.2 | 每条记录展示：时间、便便类型图标+名称、备注 | P0 |
| F3.3 | 左滑删除某条记录（带确认） | P1 |
| F3.4 | 底部「添加记录」按钮（日期预设为当前查看的日期） | P1 |
| F3.5 | 空状态：当天无记录时显示友好提示 | P1 |
| F3.6 | 导出数据：选择时间范围，生成 CSV 文件分享到微信 | P2 |

## 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF1 | 数据仅存本地 | 使用 wx.Storage，不联网 |
| NF2 | 存储容量 | 预估 10 年数据 < 1MB，无需担忧 |
| NF3 | 页面切换流畅 | 无卡顿，操作响应 < 300ms |
| NF4 | 适配 iPhone 和主流安卓 | 使用 rpx 单位，自动适配 |

## 不做什么（明确排除）

- ❌ 用户登录/注册
- ❌ 云端同步/备份
- ❌ 统计分析/图表
- ❌ 提醒/闹钟
- ❌ 多设备同步
- ❌ 分享功能
